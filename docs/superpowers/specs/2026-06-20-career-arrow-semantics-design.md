# CareerSection 화살표 의미 구분 설계

## 목표

`CareerSection`에서 과정 진행과 성능·수치 변화를 서로 다른 화살표로 표현한다.

## 표기 규칙

- `⇒`: 원인, 작업, 해결, 결과가 이어지는 과정 진행
- `→`: 시간이나 성능 지표의 이전 값에서 이후 값으로의 수치 변화

예를 들어 `폴링 ⇒ MQTT`, `설계 ⇒ 표준 제공`, `인터뷰 ⇒ 자동화`는 과정 진행이다. `21.3초→4.0초`, `1시간→5분`은 수치 변화다.

## 구현 범위

`src/components/section/CareerSection.jsx`의 사용자에게 보이는 문구만 교정한다. 주석, 다른 섹션, 컴포넌트 구조 및 스타일은 변경하지 않는다. 기존 작업 중인 프로젝트명 변경도 보존한다.

## 검증

정적 검사를 통해 과정 진행 문구에는 `⇒`, 수치 변화에는 `→`가 남아 있는지 확인하고 프로젝트 빌드를 실행한다.
